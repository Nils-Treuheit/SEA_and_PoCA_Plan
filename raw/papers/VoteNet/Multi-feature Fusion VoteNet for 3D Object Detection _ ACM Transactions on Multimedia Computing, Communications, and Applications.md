## Abstract

### Abstract

In this article, we propose a Multi-feature Fusion VoteNet (MFFVoteNet) framework for improving the 3D object detection performance in cluttered and heavily occluded scenes. Our method takes the point cloud and the synchronized RGB image as inputs to provide object detection results in 3D space. Our detection architecture is built on VoteNet with three key designs. First, we augment the VoteNet input with point color information to enhance the difference of various instances in a scene. Next, we integrate an image feature module into the VoteNet to provide a strong object class signal that can facilitate deterministic detections in occlusion. Moreover, we propose a Projection Non-Maximum Suppression (PNMS) method in 3D object detection to eliminate redundant proposals and hence provide more accurate positioning of 3D objects. We evaluate the proposed MFFVoteNet on two challenging 3D object detection datasets, i.e., ScanNetv2 and SUN RGB-D. Extensive experiments show that our framework can effectively improve the performance of 3D object detection.

## 1 Introduction

Object detection is a challenging and important task in computer vision, which needs to not only recognize the object category but also accurately locate the object’s position. At present, 3D object detection is widely used in many practical tasks \[[^5], [^22], [^23], [^30]\]. For instance, in the indoor scene, 3D object detection can help intelligent robots to plan paths or grasp objects \[[^8], [^9]\]. In the outdoor scene, 3D object detection is mainly used in automatic driving to help intelligent vehicles avoiding pedestrians and obstacles \[[^3], [^45]\]. Compared with 3D object detection, 2D object detection with **Convolutional Neural Network (CNN)** has achieved high-quality detection in complex scenes \[[^21], [^33], [^47]\]. However, since 3D object detection relies on 3D data, 2D CNN-based object detection methods have difficulty directly applying 3D object detection. A few recent methods directly operate on voxel data with 3D CNN. However, the key limitation of those methods is huge computation and memory in demand \[[^55]\]. Several other works focus on multi-view data and utilizing 2D CNN to extract features for 3D bounding boxes regression \[[^5]\]. But the multi-view images suffer from information loss, resulting in performance reduction of the detector in some cases \[[^36]\].

Recently, many researchers focus on the application of point cloud \[[^38], [^39], [^40], [^41], [^42], [^49], [^54]\]. Compared with feature learning from voxels or multi-view data, directly feature learning from raw point clouds is both computation- and memory-efficient \[[^28], [^36]\]. Owing to the properties of point clouds (e.g., disorder), CNN can not be directly applied to the point cloud. To address this issue, Qi et al. presented a point feature learning framework called PointNet \[[^31]\] for point cloud segmentation and classification, followed by many improvements and variants, especially in 3D object detection. Among them, VoteNet combines the **Deep Neural Network (DNN)** with Hough voting and only uses raw point cloud to achieve excellent performance \[[^29]\].

There are two common problems of these learning-based detection methods.

***Shallow objects***. When some shallow objects are attached to the background, it is challenging to distinguish them from each other only using the point spatial information. As shown in Figure [1](#fig1) (a), the picture is hung on the wall. When color information is available, there is a clear difference between the picture and the wall in Figure [1](#fig1) (b). Point colors can help distinguish objects from backgrounds or other adjacent objects.

![](https://dl.acm.org/cms/10.1145/3462219/asset/1200fc62-322e-4403-957a-79ac409bd24e/assets/images/large/tomm1801-06-f01.jpg)

Illustration of the importance of multi-feature information for 3D object detection from point cloud data. (a) It is difficult to find a picture in the red box without color information. (b) When the color information is added, the picture can be easily identified. (c) Some objects are missing from point cloud data. (d) In the RGB image, a sofa can be clearly seen in the red box. In fact, objects are difficult to be detected in some cases only with raw point cloud. Color and image can provide more information for 3D object detection.

***Incompletion***. Raw point data inevitably contain missing parts of some important objects in a scene, due to the measurement, algorithm errors or object occlusion. As shown in Figure [1](#fig1) (c), there should be a sofa in the red box, but it is missed in the point cloud. Fortunately, the synchronized image (the red box of Figure [1](#fig1) (d)) of a point cloud can provide the supplementary information for 3D object detection.

By fusing the colored point cloud and its RGB image, we propose a MFFVoteNet to improve the 3D object detection performance in case of incompletion and shallow objects. Our MFFVoteNet differs from the VoteNet \[[^29]\] in two designs to fuse the scene information in different modalities. First, we augment the input of VoteNet with the color information to differentiate multiple instances in 3D object detection. Then, we design an image feature module, which integrates the **Convolutional Block Attention Module (CBAM)** \[[^44]\] with the ResNet-18 backbone \[[^13]\] to encode both spatial and channel dimensions for visual attention inference. Integrating with the image feature extraction module, our method demonstrates superior 3D object detection performance in incompletion and shallow objects. Finally, **non-maximum suppression (NMS)** is an essential post-processing step in object detection. During the operation of NMS, for each category, the detected proposal with maximum score is selected to calculate **Intersection over Union (IoU)** with all other detection boxes. If the IOU of a proposal exceeds the predetermined threshold, then this proposal will be removed. In the 3D space, the IoU of a pair of boxes is not as serious as the 2D space. It is not appropriate to use the traditional 3DNMS to calculate the overlap and judge whether a proposal should be eliminated according to the IoU value. Therefore, considering the peculiarity of 3D space, we propose a **Projection NMS (PNMS)** scheme that is more suitable for 3D object detection.

To summarize, the contributions of our work are:

- We propose a novel architecture to integrate multi-modal information into 3D object detection based on VoteNet, which efficiently supplements missing information of the point cloud with the image and color features.
- We design a simple yet effective 3D NMS, namely, Projection NMS (PNMS), which can eliminate more redundant 3D proposals in post-processing by considering the particularity of 3D space.
- Extensive experiments clearly show that our proposed MFFVoteNet outperforms the baseline model as well as the state-of-the-art methods. The ablation studies further prove the effectiveness of the proposed sub-modules.

## 2 Related Works

In this section, we give a brief review on the previous work, which is related to our method.

### 2.1 3D object detection

Qi et al. \[[^31]\] propose a deep net architecture for features learning of raw point cloud, called PointNet. The advantage of PointNet is that it can process point cloud data directly, and it does not need to transform point cloud into multi-view or voxel format. PointNet has shown excellent performance and efficiency in some 3D understanding tasks without the exception of 3D object detection. Before PointNet is proposed, most 3D object detection methods use traditional methods or convolution-based learning methods. Traditional methods are challenging to fit complex semantic information, and the detection performance is weak in the evaluation of large datasets. For example, Reference \[[^35]\] is the pioneer work in 3D object detection. It can detect objects from different perspectives in images and be applied to face and car detection. Reference \[[^7]\] proposes a local voting method similar to Hough voting, which combines intensity and depth data to match scene and model features to determine the object’s location. There are many similar traditional methods, but most of them have limited detection accuracy and can only detect a subset of datasets \[[^4], [^20]\]. With the help of CNN, image object detection has achieved high performance, and this also happens in 3D object detection. Reference \[[^19]\] presents the 3D range scan data in a 2D point map and uses a single 2D end-to-end fully convolutional network to predict the proposals and the objectness confidence simultaneously. Although this method achieves high performance, it only predicts 2D proposals in 2D point map, not 3D proposals. Reference \[[^25]\] proposes to transform point cloud into a series of images and uses CNN to detect objects in 2D space. However, such a process improves efficiency, but it also loses a lot of information about the scene. Reference \[[^37]\] introduces Deep Sliding Shapes, which takes RGB-D image as input and outputs 3D proposals. Based on 3DCNN, the whole structure of Reference \[[^37]\] can be divided into two parts: the **Region Proposal Network (RPN)** and the **Object Recognition Network (ORN)**. The RPN is used to extract 3D proposals at different scales, and the ORN is responsible for 3D bounding box regression. Despite Reference \[[^37]\] showing high performance in the experiment, the prediction time is still very long (attributing to 3DCNN).

Recently benefited from the considerable success in PointNet and PointNet++, many works directly predict 3D proposals from raw point cloud data \[[^28], [^31], [^32], [^46]\]. The work in Reference \[[^30]\] employs a 2DCNN object detector to generate 2D regions and then lifts 2D regions to 3D frustum proposals. Then, an instance segmentation PointNet is used to segment the object’s point from the 3D frustum proposals. Finally, a 3D box estimation PointNet is used to predict the final detection result. Reference \[[^30]\] has achieved state-of-the-art results for both indoor and outdoor 3D object detection. However, it heavily depends on 2DCNN object detector’s performance. Reference \[[^36]\] proposes a two-stage framework for outdoor 3D object detection, called PointRCNN. The stage-1 module employs a bottom-up manner to generate a series of 3D proposals, and stage-2 combines the local spatial features and the global semantic features learned in stage-1 for 3D bounding boxes regression. Focused on indoor 3D object detection, the work in Reference \[[^29]\] proposes an end-to-end 3D object detection framework based on DNN and Hough voting, called VoteNet. A traditional Hough voting-based 2D detector employs a codebook to compute votes and retrieve offsets. Compared with the conventional method, VoteNet generates votes with a DNN-based voting module, which is more efficient and more accurate. VoteNet achieves state-of-the-art performance on two indoor datasets by using only raw point cloud as input. Reference \[[^2]\] designs an effectively shape-attentive module to capture the point local features and then also utilizes Hough voting to generate voting points. The work in Reference \[[^28]\] leverages 2D image voting to assist point cloud-based 3D object detection. Their work is based on VoteNet and further improves the performance of 3D object detection. In our work, we show another way to use multi-feature fusion to improve the accuracy of 3D object detection.

### 2.2 Multi-feature Fusion

The multi-feature fusion strategy has been extensively studied in many research fields \[[^14], [^48]\]. The work in Reference \[[^51]\] proposes a **multi-view hypergraph-based learning (MHL)** strategy that integrates click data with different visual features. For image reranking task, Reference \[[^52]\] proposes a sparse coding method based on multimodal hypergraph learning for image click prediction. Yu et al. \[[^53]\] try to solve the inconsistency between textual features and visual contents by using both visual features and click features. Similar to our work, there are many works that utilize RGB or image information combined with point clouds to obtain more semantic information \[[^10], [^17], [^24], [^26]\]. Reference \[[^10]\] employs image and point cloud features to settle classification. Ji et al. \[[^17]\] utilizes **partial differential equation (PDE)** to combine the hole repair technology with **Non-uniform Rational B-splines (NURBS)** to reconstruct a 3D scene. The UAV image and laser point cloud are used in the reconstruction process. Reference \[[^43]\] presents a person orientation estimation method based on color point cloud and achieves real-time processing. Park et al. \[[^26]\] proposes a method for aligning two colored point clouds. In our work, we use color and image information to improve 3D object detection performance on point cloud.

### 2.3 Non-maximum Suppression

**Non-maximum suppression (NMS)** is a vital post-processing technique for object detection. Recently, many researchers have proposed custom-made NMS for different detection tasks. For example, the work in Reference \[[^1]\] proposes a Soft-NMS to address the issue that some correct proposals would be missed in the traditional NMS. Compared with traditional NMS, Soft-NMS does not set the scores to zero and employs a decreasing function to change the detection scores. Soft-NMS only changes a little part of the traditional NMS and achieves noticeable improvement in COCO-style evaluation. The work in Reference \[[^11]\] proposes a modification of traditional NMS, called Seq-NMS, for video object detection. Considering the relationship between the nearby frames of video, Seq-NMS employs the high-scoring proposals from neighboring frames to enhance the weaker scores within the same video. Current state-of-the-art 3D object detection frameworks also employ NMS algorithm to eliminate redundant proposals. For the sake of distinction, we call the NMS in image object detection 2DNMS and the NMS in 3D object detection 3DNMS. In 3DNMS, we get overlap by calculating the product of length, width, and height. However, in 3D space, the proposals often very close to each other (needing-to-be-removed), while the calculated overlap is still very small (cannot be removed). By contrast, our PNMS algorithm can alleviate this situation and improves the detection accuracy.

For the image input, we employ an image encoding module to take RGB image as input. After combining the image features and the color features of point, a series of 3D proposals are predicted by the prediction module. The last module is the PNMS algorithm, which eliminates redundant proposals in the post-processing.

### 3.1 VoteNet

Our framework is designed based on VoteNet. For the convenience of narration, this subsection will give a brief introduction to VoteNet. As illustrated in Figure [2](#fig2), VoteNet includes voting module and prediction module. In the voting module, the raw point cloud (n $\times$ 3) is used as input and a series of voting points and corresponding high-dimensional features (N $\times$ (3 $+$ C)) are generated. The voting module consists of two sub-modules. In the first sub-module, it employs PointNet++ to extract the high-dimensional features of the raw point cloud and generate a series of seed points and the corresponding features. In the second sub-module, seed points and the corresponding features are processed by a **multi-layer perceptron (MLP)** to simulate Hough voting, and then the voting points and the corresponding high-dimensional features are obtained. In the prediction modules, votes are grouped and clustered to get vote clusters. Finally, an MLP is used to process the vote clusters to get the final detection results. In general, VoteNet can be expressed as follows:

$$
V o t e N e t = \left{\right. F_{p r e d i c t i o n} \left{\right. F_{v o t i n g} \left(\right. x \left.\right) \left.\right} \left.\right} ,
$$

(1)

where *x* is the raw point cloud input, $F_{v o t i n g}$ and $F_{p r e d i c t i o n}$ are the voting module and prediction module, respectively. We note that only the raw point cloud is used for 3D proposals prediction in VoteNet. In the process of data acquisition, the point cloud RGB information and the synchronized RGB images can also be acquired easily. In some scenes, only using raw point cloud data is easy to omit some important features of the object, resulting in poor detection performance. The fusion of RGB image and color information can supplement such information and improve the detection performance. Therefore, we propose a MFFVoteNet to fuse both image and point color information to obtain more scene information.

### 3.2 Multi-feature Fusion

**Color fusion.** As we mentioned in Section [1](#sec-1), in some cases, without color information may cause some objects to be missed. To overcome this problem, we integrate the point color information into VoteNet, which can be expressed as:

$$
V o t e N e t_{c o l o r} = \left{\right. F_{p r e d i c t i o n} \left{\right. F_{v o t i n g} \left(\right. x , c \left.\right) \left.\right} \left.\right} ,
$$

(2)

where the point color information is $c \in \mathbb{R}^{n \times 3}$. By involving the point color information into the VoteNet, the seed points’ features also contain color information. This can make the point voting more accurate and improves detection performance. In addition, the follow-up grouping and clustering process can also benefit from the color information transmitted by the previous network and get better detection performance.

**Image fusion.** Images express scene information in a way different from point cloud. Some objects lose their original information due to the occlusion in point cloud. We consider fusing image features into VoteNet to overcome this problem. To get the accurate objects information, we consider to set up an image encoding module. As shown in Figure [2](#fig2), the synchronized RGB image is sent to the image encoding module. To obtain superior image features, we employ ResNet \[[^13]\] and CBAM module as the feature encoding module, where CBAM is proposed by Woo et al. \[[^44]\]. And several recent works point out that the attention mechanism can improve the representation ability of DNN model. Among them, Hu et al. \[[^16]\] proposed a framework called **SEnet (squeeze and exercise networks)**. SEnet automatically learns the weight of each channel through training process and then enhances or suppresses features based on the learned weights. Compared with SEnet, CBAM introduces both spatial and channel dimensions for attention map inferring. CBAM is a lightweight module, and it can be easily integrated into other CNN frameworks. Therefore, we combine ResNet and CBAM to extract the image features. After obtaining the image features, we then integrate them into the VoteNet. As shown in Figure [2](#fig2), there are rich point cloud features in seeds, votes, and vote clusters (in the prediction module). Through experiments, we notice that combining vote features and image features could improve detection performance obviously. Specifically, image features and vote features are first combined and then grouped and clustered together in the prediction module. After integrating image features, the whole model can be expressed as:

$$
M F F V o t e N e t = \left{\right. F_{p r e d i c t i o n} \left{\right. c o n c a t \left(\right. F_{v o t i n g} \left(\right. x , c \left.\right) , F_{i m a g e} \left(\right. i \left.\right) \left.\right) \left.\right} \left.\right} ,
$$

(3)

where $F_{i m a g e} \left(\right. i \left.\right)$ is the image encoding module and *i* is the input image. As shown in Figure [2](#fig2) (c), there is a **fully connected (FC)** layer behind the last CBAM module. After the FC layer, we can get a vector of 2,560 dimensions. To match the vote features, we repeat it to $N \times 2 , 560$ dimensions.

### 3.3 Projection Non-maximum Suppression

3DNMS is proposed based on 2DNMS. As shown in Figure [3](#fig3), among the predicted proposals for the window category, the green proposals are close to the other ones with relatively low confidence, which can not be eliminated by 3DNMS effectively. Tracing to its cause, the overlap in 3DNMS is calculated as the product of the proposals length, width, and height. Under this setting, although those green proposals are close, overlap is still small. Therefore, the IoU cannot exceed the threshold. Further observing the results of 3DNMS, we found that some “thin objects” are more easily affected by the unreasonable overlap calculation, such as, window, picture, and shower-curtain. If we can select these “thin objects” and use an appropriate strategy to calculate overlap, then these proposals would be removed. Considering the unique properties of 3D space, we propose a novel method to overcome this problem, called **Projection NMS (PNMS)**. Our PNMS calculates the overlap between proposals, depending on a selective strategy. First, we choose “thin objects” according to their shapes. If the ratio of the longest edge to the shortest edge of this object is higher than a threshold, then we consider it as “thin type” object. For “thin objects,” we employ a projection function to calculate overlap. The projection function is defined as:

$$
P r o j e c t i o n \left(\right. l , w , h \left.\right) = \frac{l \cdot w \cdot h}{m i n \left(\right. l , w , h \left.\right)} ,
$$

(4)

where $l , w , h$ represent the length, height, and width of overlap, respectively. Finally, for other objects, we still use the method in 3DNMS to calculate overlap. Algorithm [1](#algorithm1) shows the pseudo code of PNMS. We use the projection function in $i o u 2 D \left(\right. \cdot \left.\right)$.

### 3.4 Model Loss

The loss function of VoteNet is a multi-task loss, including an objectness loss, a semantic classification loss, a 3D bounding box estimation loss, and a voting loss \[[^29]\]. It can be formulated as:

$$
L_{V o t e N e t} = L_{o b j - c l s} + L_{s e m - c l s} + L_{3 D - b o x} + L_{v o t i n g} .
$$

(5)

Our model adds a semantic classification loss for image encoding module:

$$
L_{i m g - c l s} = - \sum_{i = 1}^{I} \sum_{m = 1}^{M} y_{i , m} l o g \left(\right. p_{i , m} \left.\right) ,
$$

(6)

where $y_{i , m}$ is the true target label of class *m* presented at location *i* and $p_{i , m}$ is the predicted probability at location *i* of class *m*. Therefore, the loss function of MFFVoteNet can be defined as:

$$
L_{M F F V o t e N e t} = L_{V o t e N e t} + \lambda L_{i m g - c l s} ,
$$

(7)

where $\lambda$ is a weight to balance different losses. In this work, we set $\lambda$ to 0.1.

## 4 Experiment

Our experimental validation consists of five parts. First, we compare with the state-of-the-art methods for 3D object detection on two distinctive datasets: ScanNetv2 and SUN RGB-D. Second, we make the quantitative and qualitative comparison to prove the effectiveness of our method. Third, the ablation experiments are carried out to analyze the contribution of each module for performance improvement. Fourth, we further analyze the proposed method through extended experiments. Finally, we point out the limitations of our framework.

### 4.1 Dataset

We evaluate our methods on both ScanNetv2 and SUN RGB-D datasets. ScanNetv2 is a 3D reconstructed meshes dataset containing 1,513 richly-annotated of indoor scenes. Since it does not provide point clouds input and oriented 3D bounding box ground truth, we refer to Reference \[[^29]\] to generate input and ground truth. Specifically, Reference \[[^29]\] samples vertices from the reconstructed meshes as network’s input point clouds and focuses on predict axis-aligned 3D bounding boxes. ScanNetv2 also does not provide input synchronized RGB images. To overcome this problem, we employ the overhead view image instead, as shown in Figure [4](#fig4). We employ the frequently used splitting strategy mentioned in Reference \[[^29]\] to divide the whole dateset into training set and validation set. We evaluate model performance on the validation set for all 18 object categories.

![](https://dl.acm.org/cms/10.1145/3462219/asset/23a25371-6c03-496e-a723-49463333511a/assets/images/large/tomm1801-06-f04.jpg)

Examples of overhead view images in ScanNetv2 dataset.

SUN RGB-D is different from ScanNetv2: It is a partial scan RGB-D image dateset for 3D understanding tasks. The whole dataset contains 5,285 training samples and 5,050 testing samples. We evaluate our model in the testing set. SUN RGBG-D contains RGB images and we employ them as our image input. To get the point cloud data as network input, we convert the depth images to point clouds based on camera parameters. SUN RGB-D provides modal-oriented 3D bounding boxes of 37 object categories. We train and evaluate our model on the 10 most common categories following the standard evaluation protocol \[[^37]\].

### 4.2 Training details

In this article, all experiments are implemented in Python using PyTorch \[[^27]\]. We employ Adam optimizer to train our end-to-end model with a batch size of 8. For ScanNetv2 dataset, the base learning rate is set to 0.01 and we use learning rate decay of 0.1 at {80,120,160,200} epochs. For SUN RGB-D dataset, we set base learning rate to 0.001 and also use learning rate decay of 0.1 at {80,120,160,200} epochs. Besides, we employ ResNet-18 \[[^13]\] pretrained on ImageNet \[[^6]\] for processing the image input. All the images are resized to 250 $\times$ 250, which can reduce GPU memory usage. Our model is trained on a single Nvidia GTX 1080ti GPU; it takes about 6 hours for 250 epochs on ScanNetv2 and about 16 hours for 250 epochs on SUN RGB-D dataset.

### 4.5 Ablation Study

In this section, we conduct ablation experiments to evaluate the effectiveness of different sub-modules of MFFVoteNet. Table [4](#tab4) is the quantitative results of our model. Both ScanNetv2 and SUN RGB-D mAP score are included. We employ VoteNet as the baseline method, and then add the sub-modules one-by-one into the VoteNet. Observing the quantitative results in Table [4](#tab4), we found an interesting phenomenon. In SUN RGB-D, image information plays a major role in effectiveness improvement. But in the ScanNetv2 dataset, PNMS algorithm and point cloud color information contribute a lot to the mAP score increase. The reason is due to the differences between SUN RGB-D and ScanNetv2. In SUN RGB-D, many object point clouds are missing in many scenes. Images can provide missing information for these objects. In the ScanNetv2 dataset, the scene is complex and contains many “thin objects.” Color information can help distinguish different objects, while the PNMS algorithm can improve the detection performance of “thin objects.” In summary, different sub-modules of our model may have different effects on different datasets. As we expect, when all the sub-modules of our method are combined, the mAP score of our model is boosted up to the highest scores on both datasets. All these prove that our MFFVoteNet indeed improves the effectiveness of the baseline framework.

## 5 Conclusions

In this article, we show the importance of multi-feature fusion for 3D object detection. Consequently, we propose the MFF module, which integrates the multi-feature information to generate 3D proposals in point cloud data. Furthermore, to overcome the problem that traditional 3DNMS can not remove some redundant proposals, we propose the PNMS algorithm to eliminate more redundant proposals and improve the detection effectiveness. In MFF module, we employ ResNet combined with CBAM module to obtain the image feature and added semantic classification supervision. The obtained image features and point cloud features are combined together to obtain the final 3D bounding boxes. Ablation studies show that the proposed sub-modules of our MFFVoteNet indeed contribute to performance improvement. Qualitative experiments further demonstrate that our framework indeed improves the effectiveness of 3D object detection. In future work, we will focus on extending our method to other 3D understanding tasks. For example, 3D semantic segmentation task can also benefit from multi-information fusion. Multi-feature fusion can make full use of the advantages of various data and improve the model’s cognitive ability; we will explore it in the future.
